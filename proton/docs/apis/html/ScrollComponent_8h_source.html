00035 /**
00036  * A component that adds scrolling behaviour to an \c Entity.
00037  *
00038  * The name of the component is initially set to "Scroll".
00039  *
00040  * This component monitors the touch events for it's parent \c Entity and
00041  * makes the children entities of the parent move according to the events' positions.
00042  * This causes a "pannable area" effect to the contents of the parent \c Entity.
00043  *
00044  * In order to get the touch events correctly there should be a \c TouchHandlerComponent
00045  * in the parent \c Entity.
00046  *
00047  * The following named variants are used inside the component itself:
00048  * - <b>"progress2d" (Vector2):</b> the current relative position of the scroll area.
00049  *   The x coordinate of the returned value tells the horizontal and y coordinate
00050  *   tells the vertical position of the scroll area. The values are between 0 and 1.
00051  *   0 means that the scroll area is at its most left/top and 1 means that the scroll
00052  *   area is at its most right/bottom. This is a read-only variant, setting it has no
00053  *   effect. Use the "SetProgress" function object for setting this.
00054  * - <b>"swipeDetectDistance" (float):</b> How many pixels the user has to move to go
00055  *   into "scroll" mode (start scrolling). When in scroll mode the \c ScrollComponent marks
00056  *   the input events belonging to it and any children Entities should then ignore the events.
00057  * - <b>"dontScrollUntilSwipeDetected" (uint32):</b> don't move the scroll contents visually
00058  *   until scroll mode is entered.
00059  *
00060  * - Other changeable variant parms:  "friction", "maxScrollSpeed", "powerMod", "fingerTracking"
00061  *
00062  * There is one function object that this component offers:
00063  * - <b>"SetProgress"</b>: sets the relative position of the scroll area. It requires
00064  *   a Vector2 as an argument (first member of the VariantList) whose coordinate values tell the
00065  *   relative positions in the horizontal and vertical directions. The values must be
00066  *   from 0 to 1. Setting the value to (0, 0) sets the scroll area to the top left
00067  *   position. Setting the value to (1, 1) sets the scroll area to the bottom right position.
00068  */
