Detailed Description

A component that adds button behaviour to an Entity.

The name of the component is initially set to "Button2D".

This component works nicely together with TouchHandlerComponent which calls appropriate functions on the parent Entity. This component listens calls to those functions and behaves like a button.

The following named variants are used inside the component itself:

  • "onClickAudioFile" (string): the name of the audio file that is played when the button is clicked. By default is whatever is the current global default click sound. See SetDefaultAudioClickSound().
  • "disabled" (uint32): used to disable the button. 0 (the default) means the button is enabled, any other values disables it.
  • "visualStyle" (uint32): determines the button's visual style, i.e. what the button looks like when it's pressed. This needs to be one of the values of eVisualStyle. The default value is STYLE_FADE_ALPHA_ON_HOVER.
  • "buttonStyle" (uint32): determines the button's behaviour style. This needs to be one of the values of eButtonStyle. By default is whatever is the current global default button style. See SetDefaultButtonStyle().
  • "repeatDelayMS" (uint32): a delay in milliseconds during which a button is non-clickable after it has been clicked. Default is 250 milliseconds.

When the button is clicked the "OnButtonSelected" function of the parent Entity gets called. This function gets a VariantList that contains three members:

  • 0: the touch point of type Vector2
  • 1: a pointer to the Entity that was touched
  • 2: the id of the finger, type is uint32

enum Button2DComponent::eVisualStyle

Visual behaviour styles.

Enumerator:
STYLE_NONE 

No visual change.

STYLE_FADE_ALPHA_ON_HOVER 

The alpha value of the visuals is set to half of the original value.

STYLE_SCALE_DOWN_ON_HOVER 

The visuals is scaled down slightly.

STYLE_INVISIBLE_UNTIL_CLICKED 

The button is invisible until it is clicked.

After a click it stays visible for a while and then disappears again. The duration of the visibility period is the value of the repeatDelayMS variant of the component.
