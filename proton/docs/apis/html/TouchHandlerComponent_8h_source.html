00016 /**
00017  * A component that adds touch event handling behaviour to an \c Entity.
00018  *
00019  * The name of the component is initially set to "TouchHandler".
00020  *
00021  * This component is useful if you want to handle touch input events within
00022  * a given rectangular area. The area is determined by the position and size
00023  * of the parent \c Entity.
00024  *
00025  * The following named variants are used from the parent \c Entity:
00026  * - <b>"pos2d" (Vector2), "size2d" (Vector2):</b> used to determine the touch area.
00027  * - <b>"touchPadding" (Rect):</b> padding area for the touch area. This can be used
00028  *   to make the touch area larger than the area of the parent \c Entity. The default is
00029  *   (left: 20, top: 5, right: 20, bottom: 15).
00030  * - <b>"ignoreTouchesOutsideRect" (uint32):</b> defines if move events should be ignored
00031  *   when the touch position has moved outside the touch area. 0 (the default) means that
00032  *   move events outside the touch area are not ignored (i.e. they are handled). Any
00033  *   other value means that they are ignored. If this feature is enabled it basically
00034  *   means that the "OnOverEnd" function doesn't get called. As a side effect the "touchOver"
00035  *   variant also gets out of sync so the usability of this feature is questionable.
00036  * - <b>"touchOver" (uint32):</b> this value keeps track if the touch position is currently
00037  *   inside the area or not. You can read this value but don't write it!
00038  *
00039  * When events happen the following functions of the parent \c Entity get called:
00040  * - OnTouchStart - a pointer was pressed inside the area.
00041  * - OnOverStart - a pressed pointer has entered the area. This can occur when a pointer is
00042  *   pressed down inside the area (i.e. right after "OnTouchStart") or when an already
00043  *   pressed pointer is moved inside the area.
00044  * - OnOverMove - a pressed pointer moves within the area.
00045  * - OnOverEnd - a pressed pointer is moved outside the area or the pointer is released.
00046  * - OnTouchEnd - a pointer was released inside the area.
00047  *
00048  * When these functions get called they get a \c VariantList that contains four members:
00049  * - 0: the touch point of type \c CL_Vec2f
00050  * - 1: a pointer to the \c Entity that was touched
00051  * - 2: the id of the finger, type is uint32
00052  * - 3: a uint32 telling if the touch point (passed in member 0) is within the touch area.
00053  *      1 means it's inside the area, 0 means it's outside.
00054  */
