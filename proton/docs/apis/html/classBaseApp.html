void BaseApp::SetFPSLimit ( float  fps)

Limits the running rate of the main loop.

The main loop will execute (approximately) at most fps number of times per second.

Setting fps to zero removes the limitation and lets the main loop run as quickly as possible. This is also the default.

Negative values for fps are meaningless and are silently ignored.

Note:
Using this function sets an upper bound for the execution rate of the main loop. It can't be used for example to ensure a given rate of calls to Update().

boost::signal<void (VariantList*) BaseApp::m_sig_input)

Taps, clicks, and basic keyboard input.

The parameter variant list contains the following:

  • 0: the type of the event cast to a float. One of:
    • MESSAGE_TYPE_GUI_CLICK_START
    • MESSAGE_TYPE_GUI_CLICK_END
    • MESSAGE_TYPE_GUI_CLICK_MOVE
    • MESSAGE_TYPE_GUI_CLICK_MOVE_RAW
    • MESSAGE_TYPE_GUI_CHAR
    • MESSAGE_TYPE_GUI_PASTE
  • 1: the position of the event as a Vector2 if it makes sense. If the first member (the type) is either MESSAGE_TYPE_GUI_CHAR or MESSAGE_TYPE_GUI_PASTE then this is always (0, 0).
  • 2: this parameter depends on the value of the first member (the type).
    • For MESSAGE_TYPE_GUI_CLICK_START, MESSAGE_TYPE_GUI_CLICK_END, MESSAGE_TYPE_GUI_CLICK_MOVE, MESSAGE_TYPE_GUI_CLICK_MOVE_RAW this parameter defines the id of the finger used for the touch event as a uint32.
    • For MESSAGE_TYPE_GUI_CHAR this parameter contains the keycode of the pressed key as a uint32.
    • For MESSAGE_TYPE_GUI_PASTE this parameter contains the contents of the clipboard as a string.
  • 3: this parameter depends on the value of the first member (the type).
    • For MESSAGE_TYPE_GUI_CLICK_START, MESSAGE_TYPE_GUI_CLICK_END, MESSAGE_TYPE_GUI_CLICK_MOVE, MESSAGE_TYPE_GUI_CLICK_MOVE_RAW MESSAGE_TYPE_GUI_CHAR this MESSAGE_TYPE_GUI_CHAR_RAW this parameter contains the state of the control/shift/alt keys as a bitfield (uint32). Test against the eVirtualKeyModifiers to determine which keys were down at the time this msg was generated.
    • For MESSAGE_TYPE_GUI_PASTE this parameter contains the contents of the clipboard as a string.

For raw data from keyboards that give pressed/released messages.

Generally you would convert them into arcade messages.

The parameter variant list contains two members:

  • 0: the keycode of the pressed or released key as a uint32.
  • 1: VIRTUAL_KEY_PRESS or VIRTUAL_KEY_RELEASE as a uint32 depending on whether the event was a key press or release.
