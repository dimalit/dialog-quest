00019  * Audio manager handles the playing of audio.
00020  *
00021  * The \c AudioManager plays two kinds of audio files: music and sounds.
00022  * Music is meant for background music and such audio that plays possibly
00023  * for a long time. Sounds are meant for audio that usually play only for
00024  * a short time like sound effects. There can always be a maximum of one
00025  * music file playing simultaneously. There is no hard limit on how many
00026  * sounds can be playing simultaneously but the underlaying audio backend
00027  * may put some limit to this.
00028  *
00029  * The music and sound playing systems can be disabled and enabled
00030  * independently from each other.
00031  *
00032  * The actual \c AudioManager class works as a general interface for audio
00033  * managers. The \c AudioManager class by itself produces no sound
00034  * whatsoever (so it can be used to disable sound completely if instantiated
00035  * directly). In order to produce any audible sound use one of the backend
00036  * specific audio managers.
00037  *
00038  * \see AudioManagerAndroid, AudioManagerAudiere, AudioManagerBBX,
00039  * AudioManagerDenshion, AudioManagerFMOD, AudioManagerOS, AudioManagerSDL
