Detailed Description

Audio manager handles the playing of audio.

The AudioManager plays two kinds of audio files: music and sounds. Music is meant for background music and such audio that plays possibly for a long time. Sounds are meant for audio that usually play only for a short time like sound effects. There can always be a maximum of one music file playing simultaneously. There is no hard limit on how many sounds can be playing simultaneously but the underlaying audio backend may put some limit to this.

The music and sound playing systems can be disabled and enabled independently from each other.

The actual AudioManager class works as a general interface for audio managers. The AudioManager class by itself produces no sound whatsoever (so it can be used to disable sound completely if instantiated directly). In order to produce any audible sound use one of the backend specific audio managers.

See also:
AudioManagerAndroid, AudioManagerAudiere, AudioManagerBBX, AudioManagerDenshion, AudioManagerFMOD, AudioManagerOS, AudioManagerSDL

Plays an audio specified by a file name.

Returns the audio handle for the playing sound if the playing started successfully. Returns AUDIO_HANDLE_BLANK if the playing of the audio didn't start successfully. Also returns AUDIO_HANDLE_BLANK if requesting to play a sound file and sound is currently disabled or requesting to play music and music is currently disabled.

Parameters:
fNamethe file name of the audio to play.
bLoopingwhether the sound should start again when it ends, i.e. loop.
bIsMusicspecifies if this sound is the music track.
bAddBasePathadds the base path to the file name. See GetBaseAppPath().
bForceStreamingstreams the audio directly from the file and doesn't load it into memory. This flag might not be supported by all audio backends.
Returns:
The audio handle for the playing sound if the playing started successfully. Returns AUDIO_HANDLE_BLANK if the playing of the audio didn't start.
