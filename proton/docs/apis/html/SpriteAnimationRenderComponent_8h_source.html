00020  *
00021  * The name of the component is initially set to "SpriteAnimationRender".
00022  *
00023  * The following named variants are used from the parent \c Entity:
00024  * - <b>"pos2d" (Vector2):</b> specifies the position. You can move this component by changing this.
00025  * - <b>"size2d" (Vector2):</b> stores the size of the drawn surface. This is used internally,
00026  *   don't set this. Use "scale2d" instead.
00027  * - <b>"scale2d" (Vector2):</b> specifies the scale for the drawn surface. Default is (1.0, 1.0).
00028  * - <b>"rotation" (float):</b> specifies the rotation of the drawn surface. The value is in degrees.
00029  * - <b>"color" (uint32), "colorMod" (uint32), "alpha" (float):</b> these values are passed to
00030  *   \c ColorCombine() function and the result specifies a color tint for the drawn surface.
00031  *   See \c SurfaceAnim::Blit() for specifics how the tinting works. The defaults are pure opaque
00032  *   white for "color" and "colorMod" and 1.0 for "alpha". The defaults are such that the color
00033  *   tinting has no effect on the resulting image.
00034  * - <b>"visible" (uint32):</b> sets whether the surface is visible or not. 0 means not visible,
00035  *   all other values mean visible. The default value is visible.
00036  *
00037  * The following named variants are used inside the component itself:
00038  * - <b>"flipX" (uint32), "flipY" (uint32):</b> sets if the drawn frame should be flipped horizontally
00039  *   or vertically. 0 means no flipping, all other values enable the flipping. The defaults are 0.
00040  * - <b>"fileName" (string):</b> specifies the name of the sprite animation file that is used.
00041  *   See \c ResourceManager::GetSpriteAnimationSet().
00042  * - <b>"animationName" (string):</b> specifies the name of the animation this component should draw.
00043  * - <b>"phase" (float):</b> specifies the phase or progress of the animation. This must be a value
00044  *   between 0.0 and 1.0, inclusive. When "phase" is 0.0 the first frame of the animation is drawn.
00045  *   When "phase" is 1.0 the last frame of the animation is drawn. The values between 0.0 and 1.0
00046  *   map evenly to the frame numbers of the currently selected animation. The animation can be played
00047  *   back by progressively changing the value of "phase" (with \c InterpolateComponent for example).
00048  * - <b>"frameSize2d" (Vector2):</b> a read only variant that contains the unscaled frame size of the
00049  *   current animation. If there is no current animation then this value is (0.0, 0.0). See
